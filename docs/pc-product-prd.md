# VoiceAgent PC 端产品 PRD

## 1. 产品概述
- 产品名称：VoiceAgent PC 端
- 产品定位：面向桌面办公场景的语音驱动 AI 工作台
- 核心价值：用户通过键盘快捷键快速发起语音输入，系统自动识别意图并完成笔记、内容生成或任务执行
- 设计原则：
  - 键盘优先
  - 首次体验尽量短
  - 结果可见、过程可追踪
  - Action mode 先确认后执行，保证可控

## 2. 目标
- 让 PC 用户可以完全不依赖鼠标完成一次语音任务
- 让用户首次打开 60 秒内看到一次明确的 Aha Moment
- 把当前 demo 升级为可试用的桌面工作流产品
- 为后续接入开源 agent 执行链路打好基础

## 3. 目标用户
- 高频办公用户
- 需要快速记录灵感、会议要点、待办的人
- 需要通过语音发起任务执行的人
- 习惯使用键盘快捷键提升效率的桌面用户

## 4. 核心场景
- 语音记笔记
- 语音生成内容摘要/草稿
- 语音发起动作并让 agent 执行
- 查看历史记录、结果、执行日志

## 5. 版本范围

### V1 必做
- PC 端快捷键录音
- 录音结束后自动进入处理流程
- 首次进入的新手引导
- 场景化示例任务
- Action mode 的计划展示与人工确认
- 基础历史记录与日志
- 失败提示和重试

### V1 不做
- 移动端
- 复杂多 agent 协作
- 全自动高风险执行
- 高级权限体系
- 团队协作功能

## 6. 用户流程

### 6.1 首次进入
- 检测用户是否首次打开
- 展示简短引导卡片
- 让用户选择一个主场景：
  - 记笔记
  - 生成内容
  - 执行动作
- 提示最少配置：
  - 麦克风权限
  - API 配置状态
- 提供一个默认可试的语音示例
- 用户完成一次录音后展示结果，形成 Aha Moment

### 6.2 日常使用
- 用户按快捷键开始录音
- 再次按快捷键结束录音
- 系统自动处理语音
- 若识别为 note/publish，展示结果
- 若识别为 action，展示计划并等待确认
- 用户确认后执行
- 完成后写入历史

### 6.3 Action flow
- 语音输入
- 转成结构化 action plan
- 展示解析结果
- 用户确认
- 调用 agent 执行
- 展示执行步骤、状态与结果
- 失败可重试或取消

## 7. 交互设计

### 7.1 键盘快捷键
- `Ctrl/Cmd + Alt + V`：按下录音，松开结束
- `Ctrl/Cmd + Alt + K`：打开快捷操作面板
- `Ctrl/Cmd + Alt + Enter`：确认当前 action
- `Esc`：取消录音或中断当前任务
- `Ctrl/Cmd + Alt + /`：查看快捷键说明

### 7.2 首页布局
- 顶部状态栏
  - 当前模式
  - 麦克风状态
  - 快捷键提示
  - 设置入口
- 中央主操作区
  - 录音按钮
  - 快捷键提示
  - 实时转写
- 下方结果区
  - 笔记
  - 摘要
  - Action plan
  - 执行状态
- 右侧或下方历史区
  - 最近任务
  - 执行日志
  - 可回看记录

### 7.3 Aha Moment 设计
- 首次进入直接给一个可执行示例
- 示例要真实、短、具体
- 结果展示要明确告诉用户“你刚刚完成了什么”
- 成功后给下一步建议，降低流失

## 8. 功能需求

### 8.1 新手引导
- 首次打开自动出现
- 支持跳过
- 支持回看
- 只做 1 到 3 步
- 重点解释“怎么开始”和“能得到什么”

### 8.2 语音输入
- 支持快捷键开始/结束
- 支持麦克风权限检测
- 支持实时转写
- 支持缓冲期内继续输入
- 支持录音超时自动停止

### 8.3 意图识别
- 识别三类意图：
  - note
  - publish
  - action
- 支持手动模式覆盖自动识别
- 识别结果可展示在 UI 上

### 8.4 Action mode
- 解析自然语言为结构化 action
- 显示 action 类型、标题、描述、关键字段
- 显示执行计划
- 提供确认/取消
- 执行过程可视化
- 执行结果回写历史

### 8.5 Agent 执行层
- 预留 agent adapter 接口
- 支持接入 openclaw 或其他开源 agent
- 支持任务计划输入
- 支持执行日志输出
- 支持执行结果回传
- 支持失败恢复

### 8.6 历史记录
- 保存最近语音输入
- 保存 note/publish/action 历史
- 支持查看结果和时间
- 支持清空历史

### 8.7 设置
- 麦克风权限状态
- API 配置状态
- 快捷键说明
- agent 连接状态
- 新手引导重置

## 9. 数据与状态设计

### 核心状态机
- `idle`
- `recording`
- `buffering`
- `transcribing`
- `analyzing`
- `planning`
- `awaiting-confirmation`
- `executing`
- `done`
- `failed`

### 核心数据对象
- Transcript
- Intent
- Action plan
- Execution result
- History record
- User config

## 10. 埋点与指标

### 北极星指标
- 首次成功语音任务完成率

### 关键指标
- 首次打开到首次成功的时间
- 快捷键使用率
- 录音成功率
- Action 确认率
- Action 执行成功率
- 任务完成耗时
- 次日留存率

## 11. 风险与对策

### 风险 1：新手引导太长，用户流失
- 对策：只保留最少步骤，先让用户成功一次

### 风险 2：快捷键冲突
- 对策：使用 `Ctrl/Cmd + Alt + V` 这类较少冲突的组合，并允许后续配置

### 风险 3：Action 执行不可控
- 对策：高风险操作必须确认，支持中断和回滚提示

### 风险 4：agent 接入不稳定
- 对策：先做适配层，不把前端和具体 agent 强绑定

### 风险 5：用户不理解产品价值
- 对策：把 Aha Moment 设计成“说一句话，立刻完成一件事”

## 12. 里程碑

### 里程碑 1：桌面输入可用
- 快捷键录音可用
- 转写可用
- 页面状态清晰

### 里程碑 2：首次体验闭环
- 新手引导上线
- 示例任务可跑通
- 首次成功体验可见

### 里程碑 3：Action 真执行
- 结构化 action plan
- agent adapter 接入
- 至少 1 到 3 个真实动作可执行

## 13. 验收标准
- 用户可在无鼠标情况下完成一次语音输入
- 首次进入能在 60 秒内完成一次有效结果
- Action mode 能从语音到执行形成闭环
- 历史和日志可追踪
- 页面在桌面浏览器上稳定运行
